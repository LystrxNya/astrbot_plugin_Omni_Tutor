# 🎓 OmniTutor (全知导师)

**“主人的知识，由可琳来守护。不论是深奥的定理还是零碎的笔记，我都会为您打理得井井有条哦♪”**

OmniTutor 是一款基于 **AstrBot** 框架开发的深度学术科研与个人知识管理插件。它不仅是一个 RAG（检索增强生成）工具，更是一个具备**动态认知追踪**、**异构知识图谱**与**多模态视觉解析**能力的智能学习中枢。

本插件内置了“治愈系女仆导师 · 可琳”的人格设定，旨在将枯燥的资料录入转化为有温度的互动式学习体验。

---

## ✨ 核心特性

### 🧠 双脑协同架构 (Dual-Brain Engine)
* **快脑 (Fast Model)**：基于 `qwen-turbo/flash`，负责高并发的文档打标、OCR 语义清洗与基础意图识别，响应极其迅速。
* **慢脑 (Reasoning Model)**：基于 `deepseek-v3/qwen-plus`，负责复杂的学术意图裂变、认知障碍诊断以及长篇资料的深度精读重构。

### 🔍 进化级 RAG 检索流
* **意图裂变**：慢脑将用户提问拆解为多个学术概念与教学意图（定义、应用、练习、拓展）。
* **双路召回**：
    1.  **图谱路由**：通过 Inverted Index（倒排索引）精准定位相关学术节点。
    2.  **语义盲捞**：基于向量相似度进行全库语义搜索，确保不遗漏上下文。
* **安全截断与重排**：内置 `MAX_CANDIDATES` 保护阀，并使用 BGE-Reranker 进行交叉注意力评分，只喂给大模型最高纯度的知识切片。

### 👁️ 多模态视觉引擎 (Omni-Vision)
* **高精度 OCR**：支持 LaTeX 公式完美转录，严谨区分学术符号。
* **逻辑图形描述**：不仅识别文字，还能观察并描述几何图形、物理受力图、电路图等结构，弥补大模型视觉理解的短板。
* **智能缝合**：自动拦截纯图片输入，开启 8 秒等待窗口进行图文缝合投喂。

### 📈 动态认知内窥镜
* **掌握度追踪**：根据交互深度（识记/应用/综合）动态计算用户对特定概念的掌握度。
* **遗忘曲线模拟**：内置时间衰减因子，真实反映知识在脑中的存留状态。
* **阻断性诊断**：自动识别用户的“认知受阻”点（概念崩塌或逻辑卡壳）。

## 🛠️ 指令系统

### 1. 知识捕获
| 指令 | 功能说明 |
| :--- | :--- |
| `/学习` | 开启主库录入，支持图片、PDF 及 Office 文档，具备 5 秒防抖打包机制。 |
| `/精读` | 深度思考模式，针对长文档生成详尽的 Markdown 学术笔记。 |
| `/ocr` | 纯文字提取模式，支持多图并发扫描与 LaTeX 重构。 |
| `/重要录入` | 独立存储碎片化信息（账号、密码、考点等），物理隔离于主学术库。 |

### 2. 图谱手术刀 (Management)
| 指令 | 功能说明 |
| :--- | :--- |
| `/自清洗 [可选文件名]` | **全局/靶向清洗**。消除同义词冗余，重连物理切片与逻辑节点的羁绊。 |
| `/合并概念 [旧] [新]` | 手动强行合并两个学术节点。 |
| `/重塑概念 [名称]` | 呼叫慢脑对属于该概念的所有切片进行重写、去重并重新打标。 |
| `/撤销` | 快速回滚最后一次入库操作。 |

### 3. 诊断与反馈
| 指令 | 功能说明 |
| :--- | :--- |
| `/诊断` | 生成基于遗忘曲线的个人认知报告长图。 |
| `/图谱` | 实时渲染当前知识库的扁平化概念分布图。 |
| `/取消` | **全链路紧急刹车**。立即中止 OCR、学习或生成的异步进程，释放系统资源。 |

## 🚀 技术亮点

1.  **拦截探针机制**：在所有高耗时循环（并发 OCR、打标、Map-Reduce 洗稿）中植入 `is_aborted` 探针，确保 `/取消` 指令能即时切断后台 API 消耗。
2.  **聚类避嫌算法**：在自动归一化时采用物理隔离策略，防止新词与自身匹配产生 1.0 相似度死锁，大幅提升图谱纯净度。
3.  **内存向量缓存**：挂载全局内存级向量 Cache，对重复出现的学术名词实现纳秒级检索，告别 O(N²) 嵌入请求风暴。
4.  **智能拼图缝合**：当检索到的切片存在“边界损耗”时，系统会自动回表查询上下文进行动态缝合。

## ⚙️ 配置说明

您需要在 AstrBot 的插件配置中填入以下密钥：
* `dashscope_api_key`: 阿里云百炼 API Key。
* `siliconflow_api_key`: 硅基流动 API Key（用于高性能 Embedding 与 Rerank）。
* `fast_model`: 推荐 `qwen-plus` 或 `qwen-turbo`。
* `reasoning_model`: 推荐 `deepseek-v3`。
